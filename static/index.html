Welcome to the Simple FastAPI Portfolio API Demo!

This application demonstrates the essential features of FastAPI.

Explore the API:

  • Interactive API Docs (Swagger UI): /docs - Try out the API endpoints directly from your browser.
  • Alternative API Docs (ReDoc): /redoc - View alternative, more static documentation.

FastAPI Features Demonstrated:

Interact with the API via the /docs page to see these in action:

  • Automatic Docs: Generated automatically from your code (check /docs and /redoc). Docstrings and type hints contribute to this.
  • Path Parameters: Used in GET /portfolio/projects/{project_id} to identify a specific project. Try fetching project ID 1 or 2.
  • Query Parameters: Used in GET /portfolio/projects for pagination (e.g., /portfolio/projects?skip=1&limit=1). Try changing the skip and limit values in the docs.
  • Request Body & Pydantic Models: Used in POST /portfolio/projects to create a new project. The API expects a JSON body matching the ProjectCreate schema defined in models.py. Try creating a project via the docs - invalid data will result in a 422 error.
  • Response Models: Used in all endpoints (e.g., response_model=models.Project) to define the exact structure and types of the JSON response, ensuring consistency and filtering sensitive data if needed. Observe the structured JSON output in the docs.
  • Data Validation: Pydantic models (models.py) automatically validate path parameters, query parameters, and request bodies based on type hints and constraints (like min_length, gt=0). Try providing invalid data (e.g., a short description when creating a project) to see the validation error response.
  • Dependency Injection: A simple dependency (get_common_parameters) is defined in routers/portfolio.py and shown how it *could* be injected using Depends (though not actively used in the final logic here for simplicity). This is crucial for managing database connections, authentication, etc.
  • Async Support: Route handlers are defined with async def, allowing FastAPI to handle requests concurrently for better performance, especially with I/O operations.
  • Error Handling: HTTPException is used in GET /portfolio/projects/{project_id} to return a 404 Not Found error if the ID doesn't exist. Automatic 422 Unprocessable Entity errors are returned for validation failures.
  • APIRouter: The endpoints are organized using APIRouter in routers/portfolio.py, making the codebase modular. All routes are prefixed with /portfolio.
  • Static Files: This HTML page (index.html) is served as a static file.
